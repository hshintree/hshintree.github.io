Methodology & Credits

─── Optimisation Problem ───────────────────────────────────

  max  (μ_eff − rf)ᵗ w  −  γ · wᵗ Σ w
  s.t. Σwᵢ = 1,   0 ≤ wᵢ ≤ max_wt

  γ (risk aversion): higher γ tilts the portfolio toward minimum
  variance, sacrificing expected return to reduce volatility.

─── Expected Returns μ ────────────────────────────────────

  Step 1: EWMA of past log-returns, half-life = 252 trading days.
          μᵢ_EWMA = exponential average of daily returns, more weight
          on recent observations, annualised × 252.

  Step 2: James-Stein cross-sectional shrinkage toward the equal-
          weighted mean:
            μ_eff = (1 − α) · μ_EWMA + α · mean(μ_EWMA)
          α = shrinkage slider (0% = raw EWMA, 100% = all assets get
          the same μ). High shrinkage reduces sensitivity to noisy
          return estimates, important when history is short.

─── Covariance Σ ──────────────────────────────────────────

  EWMA of the outer product of demeaned returns, half-life = 21
  trading days (captures recent regime shifts quickly). Annualised
  × 252. A small ridge ε·I is added to ensure positive
  definiteness.

─── Solver ───────────────────────────────────────────────────

  Projected gradient ascent. Each step: gradient ascent on the
  Lagrangian, then project onto the simplex ∩ box (Condat 2016,
  binary search on the dual variable θ, O(N log N)).

─── Walk-Forward Backtest ─────────────────────────────────

  On each rebalance date: fit μ and Σ on all historical data
  available up to that date (expanding window), solve for w*, hold
  until the next rebalance. Performance is always out-of-sample;
  the optimiser never sees future prices.

  Benchmarks: SPY (buy-and-hold) and 1/N (equal-weight, rebalanced
  on the same schedule).

─── Output Panels ───────────────────────────────────────────

  Current Allocation: the portfolio weights from the most recent
    rebalance, i.e. what the model recommends holding today.

  Portfolio Value: cumulative return of the Markowitz portfolio vs
    SPY and 1/N over the backtest window, normalised to 1.0 at start.

  Weights Heatmap: how allocations shifted at each rebalance. Navy
    = 0% (no allocation), amber = max-weight cap. Colour scale is
    linear between the two.

  Forecast Accuracy: at each rebalance the model's μ and σ
    predictions are compared against what actually happened over the
    next period. See the column key above the table.

  IC Over Time: Spearman rank correlation between the vector of
    predicted returns and the vector of realized returns across all
    assets at each rebalance. Positive IC = the ordering was correct
    (assets the model ranked higher actually did better).

  Realized vs Predicted μ: scatter of every (asset, rebalance-date)
    pair. Points along the diagonal = perfect forecasts. Hover to see
    which asset and rebalance period each point represents.

─── Limitations & Interpretation ───────────────────────────

  Return forecasting is hard. IC values and Pearson r near zero
  confirm that EWMA μ has almost no out-of-sample predictive power
  at monthly horizons. This is expected: short-horizon stock returns
  are dominated by noise, not the slow-moving signal captured by
  historical averages.

  What this means in practice:
  • High shrinkage (α → 100%) collapses all predicted returns to
    the same value, reducing the problem to pure minimum-variance.
    The weights then reflect only correlations and volatilities,
    not any view on which stocks will outperform.
  • Low shrinkage (α → 0%) lets raw EWMA returns drive the
    allocation. Because those estimates are noisy and non-stationary,
    the portfolio becomes erratic and concentrated in recent winners.
  • Covariance forecasts are better calibrated than return forecasts
    (vol ratio ~1.1x here), but the short 21-day EWMA half-life
    amplifies recent turbulence, making the portfolio more risk-averse
    than a long-run view would justify, i.e., under-allocating to
    assets that simply had a volatile patch.

  If you want more robust results, consider: factor-model expected
  returns (Fama-French, CAPM), Ledoit-Wolf covariance shrinkage,
  transaction-cost-aware optimisers, or Black-Litterman to blend
  prior views with equilibrium returns.

─── Data & Infrastructure ──────────────────────────────────

  Price data: Alpaca Markets API, adjusted daily closes (IEX feed).
  API keys are stored server-side in a Cloudflare Worker; they are
  never sent to or stored in the browser.

  Caching: fetched data is cached in Cloudflare KV (shared across all
  visitors) and in your browser's localStorage. If you or any other
  user has already fetched a symbol over the same date range, the
  data is served from cache instantly with no API call.

  All computation (EWMA, optimisation, backtest, accuracy stats) runs
  entirely in your browser. No data leaves your machine.
